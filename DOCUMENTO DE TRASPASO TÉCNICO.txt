Proyecto: HarryNine Bot V14
1. RESUMEN GENERAL DEL PROYECTO
¿Qué hace este bot?

HarryNine es un bot de Telegram orientado al análisis deportivo de fútbol. Su objetivo principal es detectar oportunidades de apuesta pre-match y live usando datos estadísticos de API-Football.

El bot analiza partidos, genera recomendaciones de mercados, asigna score, riesgo, probabilidad estimada, cuota mínima recomendada, guarda las jugadas para seguimiento y posteriormente genera reportes de rendimiento.

El enfoque actual del proyecto es construir un sistema tipo tipster cuantitativo que no solo sugiera picks, sino que también mida su efectividad real con resultados históricos.

Propósito principal

El bot sirve para:

Consultar fixtures confiables por región.
Analizar partidos pre-match.
Analizar partidos live.
Detectar oportunidades fuertes.
Ordenar picks por score y riesgo.
Guardar automáticamente las jugadas sugeridas.
Actualizar resultados automáticamente.
Generar reportes diarios, semanales y mensuales.
Construir una base histórica para evaluar mercados, ligas, score, riesgo y efectividad.

2. ARQUITECTURA Y FLUJO DE FUNCIONAMIENTO
Flujo general de arranque

El bot arranca ejecutando:

python bot.py

El flujo general es:

1. Carga variables de entorno desde .env.
2. Define constantes globales.
3. Define listas de ligas configuradas.
4. Define funciones utilitarias.
5. Define funciones de API.
6. Define funciones de análisis pre-match.
7. Define funciones de análisis live.
8. Define funciones de guardado y actualización de resultados.
9. Define comandos de Telegram.
10. Registra los CommandHandler.
11. Ejecuta app.run_polling().

El bot imprime algo parecido a:

🤖 HarryNine V14 ejecutándose...
Cómo recibe mensajes

Usa python-telegram-bot.

Cada comando se registra así:

app.add_handler(CommandHandler("comando", funcion))

Ejemplo:

app.add_handler(CommandHandler("resumen", resumen))

Cuando el usuario escribe:

Telegram envía el evento, python-telegram-bot lo procesa y llama a la función async correspondiente.

5. LÓGICA DE NEGOCIO Y CRITERIOS
Filosofía del bot

La decisión de diseño principal es:

Calidad > cantidad.

El bot no debe forzar picks si no hay valor.

Score

El score va de 0 a 10.

Interpretación:

10/10 = señal muy fuerte
9/10 = élite
8/10 = fuerte
7.5/10 = top mínimo recomendado
7/10 = aceptable, pero actualmente se considera más débil
<7 = no recomendado
Riesgo

El riesgo va de 0 a 10.

Interpretación:

1-2 = bajo
2-4 = moderado
4-6 = alto
>6 = evitar
Probabilidad

Probabilidad estimada interna según reglas del modelo.

No es garantía real. Debe compararse luego con resultados y cuotas.

Actualmente muchas cuotas son internas/estimadas:

cuota_minima
cuota_justa
cuota

Se detectó que no son cuotas reales de casa de apuesta.

Recomendación:

API-Football = cerebro estadístico
Odds API = validación final de cuota
Mejoras planeadas
1. Módulo de bank management

Comandos sugeridos:

/bank
/stake
/plan_dia
/plan_agresivo
/retroescalera

Objetivo:

Administrar bank.
Recomendar stake.
Simular estrategia diaria.
Controlar riesgo.

Advertencia:

No garantizar 10% diario. Solo simular o sugerir con control de riesgo.
2. Retro escalera

Estrategia propuesta:

Stake inicial pequeño
3 a 5 escalones
Score mínimo 8.5+
Cuotas 1.25 a 1.45
Detener al lograr objetivo
No reiniciar si falla

Debe ser módulo separado dentro del mismo bot.

3. Autoaprendizaje

La base del autoaprendizaje será:

picks guardados + resultados actualizados

Se debe analizar:

Efectividad por mercado.
Efectividad por liga.
Efectividad por score.
Efectividad por riesgo.
Efectividad live vs prematch.
ROI estimado si se integran cuotas reales.
4. Árbitros para tarjetas

Para mejorar mercado de tarjetas, agregar análisis de árbitro:

Promedio de amarillas.
Promedio de rojas.
Tendencia tarjetero.
Liga.
Historial.
5. Odds reales

Integrar cuotas reales solo cuando:

score >= 7.5

No consultar cuotas para todos los partidos para no gastar créditos.

6. Mejorar PDFs

El PDF diario ya funciona, pero se puede mejorar:

Mejor diseño.
Tablas.
Separar prematch/live.
Resumen por mercado.
Resumen por país.
Ranking de mejores picks.
Paginación profesional.
Exportar CSV/Excel.
Reglas importantes para el nuevo asistente
No modificar todo el bot de golpe.
Hacer cambios pequeños y probar.
Mantener el tracking limpio.
No guardar /detalle automáticamente.
Evitar contaminar dataset con picks débiles.
Mantener /analizar como TOP 1.
Mantener /detalle como análisis profundo.
Mantener /resumen como PDF.
Priorizar calidad sobre cantidad.
No forzar picks si no hay valor.

Nota final para el nuevo asistente

Este proyecto no es solo un bot de picks. Es un sistema de análisis, tracking y evolución estadística.

La lógica correcta es:

Fixtures
→ Análisis estadístico
→ Score/riesgo
→ Pick recomendado
→ Guardado
→ Resultado real
→ Reporte
→ Medición
→ Ajuste del modelo

La visión del proyecto es convertir HarryNine en un sistema tipster con:

análisis pre-match
análisis live
gestión de bank
retro escalera
tracking histórico
reportes PDF
autoaprendizaje

La prioridad inmediata debería ser:

1. Estabilizar /top_manana y /elite_manana.
2. Terminar /resumentop y /resumentoplive.
3. Revisar guardado por fuente y tipo.
4. Optimizar PDF diario.
5. Integrar cuotas reales solo para picks fuertes.
